# 보세전시장 민원응대 챗봇 데이터

법제처 국가법령정보센터의 현행 법령과 관세청 공식 자료를 기반으로 한 보세전시장 민원응대 챗봇 시스템입니다.

## 프로젝트 구조

```
bonded-exhibition-chatbot-data/
├── config/
│   ├── system_prompt.txt          # 챗봇 시스템 프롬프트
│   └── chatbot_config.json        # 챗봇 설정 (페르소나, 카테고리, 연락처)
├── data/
│   ├── faq.json                   # FAQ 데이터셋 (7개 항목)
│   ├── legal_references.json      # 법령 근거 데이터
│   └── escalation_rules.json      # 에스컬레이션 규칙 (5개 조건)
├── templates/
│   └── response_template.json     # 답변 포맷 템플릿
├── src/
│   ├── chatbot.py                 # 메인 챗봇 로직
│   ├── classifier.py              # 질문 의도 분류기 (10개 카테고리)
│   ├── response_builder.py        # 답변 생성기
│   ├── escalation.py              # 에스컬레이션 판단 로직
│   ├── validator.py               # 확인 질문 관리
│   └── utils.py                   # 유틸리티 함수
├── tests/
│   ├── test_chatbot.py            # 통합 테스트 (19개)
│   ├── test_classifier.py         # 분류기 테스트 (14개)
│   ├── test_response_builder.py   # 답변 생성기 테스트 (9개)
│   ├── test_escalation.py         # 에스컬레이션 테스트 (10개)
│   └── test_validator.py          # 검증기 테스트 (9개)
├── simulator.py                   # 챗봇 시뮬레이터
├── requirements.txt
└── README.md
```

## 시작하기

### 설치
```bash
pip install -r requirements.txt
```

### 테스트 실행
```bash
python -m pytest tests/ -v
```

### 챗봇 시뮬레이터

시뮬레이터는 3가지 모드를 지원합니다:

```bash
# 대화형 모드 - 터미널에서 직접 질문/답변
python simulator.py

# 자동 테스트 - 미리 정의된 10개 시나리오 자동 검증
python simulator.py --test

# 단일 질문 - 특정 질문 하나만 테스트
python simulator.py --query "보세전시장이 무엇인가요?"
```

## 핵심 법적 근거

| 법령 | 조문 | 내용 |
|------|------|------|
| 관세법 | 제190조 | 보세전시장 정의 |
| 관세법 | 제161조 | 견본품 반출 (세관장 허가) |
| 관세법 시행령 | 제101조 | 판매용품의 면허전 사용금지 |
| 관세법 시행령 | 제102조 | 직매된 전시용품의 통관전 반출금지 |
| 관세청 고시 | 제2026-15호 | 보세전시장 운영에 관한 고시 |

## 질문 분류 카테고리 (10개)

1. 제도 일반 / 2. 특허/운영 / 3. 반입/반출 / 4. 전시/사용 / 5. 판매/직매
6. 견본품 / 7. 시식용 식품 / 8. 서류/신고 / 9. 벌칙/제재 / 10. 담당기관 문의

## 에스컬레이션 규칙

다음 경우 자동으로 사람 상담 연결을 안내합니다:
- 현장 판매 후 즉시 인도 여부 → 관할 세관
- 시식/검역/식약처 요건 복합 → 고객지원센터(125)
- 특허 적정성/제재 가능성 → 보세산업과
- 유권해석 요구 → 보세산업과
- UNI-PASS 전산 이슈 → 기술지원센터(1544-1285)
